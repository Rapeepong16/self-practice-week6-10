ทำไมต้องมีการเก็บข้อมูลฝั่งผู้ใช้ (Client Side)?
เวลาเราเล่นเว็บต่าง ๆ เช่น Facebook, Lazada, Shopee หรือ Google – เว็บพวกนี้จะต้องจดจำเรา เช่น:
- ใครเป็นคนล็อกอินอยู่
- เราเลือกธีมสีเข้มหรือสว่าง
- สินค้าที่เราใส่ไว้ในตะกร้า
- แบบฟอร์มที่เรากรอกค้างไว้
- การตั้งค่าภาษา, ภูมิภาค
หากไม่มีระบบพวกนี้ ข้อมูลทั้งหมดจะหายทุกครั้งที่เราเปลี่ยนหน้า หรือปิดเว็บ ซึ่งจะทำให้ ประสบการณ์ใช้งานแย่มาก และ ไม่ปลอดภัย

Token(เก็บตอนเวลาเรา Login)
-Access Token
-Refresh Token (จำได้นานกว่าเเละเป็นตัวไปแรก Access Token)


Cookies:
- ใช้เพื่อจัดการ การเข้าสู่ระบบ (Login) แบบปลอดภัย
- ใช้เพื่อให้เว็บรู้ว่าเราเป็นใคร แม้เราจะ ปิดหน้าเว็บไปแล้วกลับมาใหม่
- ใช้ในการติดตามผู้ใช้งาน เช่น สำหรับโฆษณา / วิเคราะห์พฤติกรรม
- เก็บข้อมูลฝั่ง client ขนาด ≤ 4 KB
- ฝั่ง server จะขอให้สร้าง Cookies --> Client (ผ่าน http response)
- first party --> เก็บจากหน้าเว็บที่เราใช้งานอยู่ / third party --> เก็บไปใช้เว็บอื่น
!ข้อควรระวัง:
- ข้อมูลที่เก็บไว้จะถูกส่งไปกับทุกการโหลดหน้าใหม่ → เสี่ยงเรื่องความปลอดภัยและความเร็วถ้าใส่ข้อมูลมากไป
- ถ้าไม่ตั้ง HttpOnly หรือ Secure อาจถูกขโมยโดย JavaScript ได้ (เสี่ยง XSS)

How to Create Cookies
set-cookie: cookie-name=cookie-value

path เวลาสร้าง cookie เราเริ่มสร้างจาก path ไหนถ้าเรียก path นั้นมันส่ง cookie หมด
รวมถึง sub directory ที่ถูกเรียกด้วย


Local Storage:
- เก็บข้อมูลที่ต้องการให้ จำไว้ถาวร จนกว่าจะลบเอง เช่น การตั้งค่าของผู้ใช้
- ไม่ส่งไปที่เซิร์ฟเวอร์ จึงช่วยให้เว็บ ทำงานเร็วขึ้น และลดโหลดเซิร์ฟเวอร์
!ข้อควรระวัง:
- เข้าถึงได้จาก JavaScript → อย่าเก็บข้อมูลสำคัญ เช่น รหัสผ่าน, Token
- ข้อมูลอยู่ไปเรื่อย ๆ จนกว่าจะลบ → ต้องเคลียร์บ้างเพื่อไม่ให้เปลืองพื้นที่

Session Storage:
- เหมาะกับข้อมูลที่ใช้แค่ “ชั่วคราว” เช่น ตอนเปิดเว็บอยู่ในแท็บเดียว
- ปลอดภัยกว่าบ้าง เพราะข้อมูลจะหายเมื่อปิดแท็บนั้น
ข้อควรระวัง:
- ข้อมูลจะหายเมื่อปิดแท็บ → อย่าใช้เก็บอะไรที่ต้องจำไว้นาน
